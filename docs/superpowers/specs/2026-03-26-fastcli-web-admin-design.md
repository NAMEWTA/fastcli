# fastcli web 管理后台设计规格

## 1. 背景与目标

当前 `fastcli` 的配置管理以终端和 JSON 文件编辑为主。目标是新增 `fastcli web` 入口，在本机自动启动一个管理后台页面，用可视化方式管理 `~/.fastcli/config.json`。

本规格聚焦首版（v1）可落地能力，不扩展到拖拽编排、远程协作或多用户系统。

## 2. 范围定义

### 2.1 In Scope（v1）

- 命令：`fastcli web`
- 启动行为：
  - 自动启动本地服务
  - 自动打开浏览器
  - 终端打印 URL
- 鉴权：
  - 启动时生成一次性口令
  - 在终端打印口令
  - 页面输入口令后进入后台
- 配置目标：`~/.fastcli/config.json`
- 页面布局：传统三段式管理后台（顶部 / 左侧 / 中间主区域）
- 首页：全局配置总览（`aliases`、`providers`、`credentials`、`workflows` 四块）
- 编辑：仅编辑已有条目（不支持新增/删除）
- 编辑交互：列表 + 右侧抽屉详情
- 抽屉编辑：双模式切换（默认表单，可切 JSON）
- 顶部主操作：保存全部 / 导入 / 导出
- 保存策略：手动保存（保存全部）
- 校验策略：严格阻断（有错误则禁止保存）
- JSON 模式校验：点击“校验”后统一错误列表
- 导入导出：支持导入 + 导出
- 导入策略：全量覆盖当前配置
- 导入保护：普通二次确认弹窗（确认覆盖）
- 总览指标：数量 + 最近更新时间
- 后续删除能力优先模块：`aliases`

### 2.2 Out of Scope（v1 不做）

- 新增条目、删除条目
- 拖拽式工作流设计器
- 实时协作编辑
- 远程配置源（非本机配置路径）

## 3. 方案对比

### 3.1 方案 A（采用）

内嵌静态 Web + CLI 本地 API。

- `fastcli web` 在当前 CLI 进程体系内启动本地服务
- React 前端构建产物由 CLI 托管
- 前端通过本地 API 读取/校验/保存配置

优点：

- 与现有 CLI 集成紧密，单体交付
- 用户上手成本低（一个命令启动）
- 复用现有配置校验逻辑和类型语义

缺点：

- CLI 与 Web 生命周期耦合

### 3.2 方案 B（未采用）

独立前后端工程。

优点：扩展性高。
缺点：工程复杂度和维护成本高，不符合 v1 目标。

### 3.3 方案 C（未采用）

纯服务端模板页面（非 React）。

优点：实现快。
缺点：交互能力不足，不适配抽屉双模式编辑要求。

## 4. 信息架构与页面结构

### 4.1 视觉方向

- 风格：专业企业风
- 结构：
  - 顶部导航栏
  - 左侧导航栏
  - 中间主内容区

### 4.2 左侧导航（混合分组）

- 上半区按任务：
  - 配置总览
  - 配置校验
  - 导入配置
  - 导出配置
- 下半区按类型：
  - Aliases
  - Providers
  - Credentials
  - Workflows

### 4.3 顶部导航主操作

- 保存全部
- 导入
- 导出

## 5. 关键交互流程

### 5.1 启动与鉴权流程

1. 用户执行 `fastcli web`
2. CLI 启动本地服务并自动打开浏览器
3. CLI 在终端打印：
   - 访问 URL
   - 一次性口令
4. 页面初始为口令输入页
5. 验证通过后进入管理后台主界面

### 5.2 编辑与保存流程

1. 用户在列表选中条目
2. 右侧抽屉打开详情
3. 默认表单模式编辑，必要时切到 JSON 模式
4. 在 JSON 模式点击“校验”
5. 系统统一展示错误列表
6. 用户修复错误
7. 点击“保存全部”
8. 若存在任意错误，阻断保存并提示
9. 全部通过后写入 `~/.fastcli/config.json`

### 5.3 导入流程（全量覆盖）

1. 点击“导入”并选择 JSON 文件
2. 展示确认弹窗（普通二次确认）
3. 用户确认“覆盖”
4. 以导入内容全量替换当前工作态
5. 触发统一校验
6. 错误则禁止保存，提示修复
7. 用户点击“保存全部”完成落盘

### 5.4 导出流程

1. 点击“导出”
2. 导出当前工作态配置为 JSON 文件

## 6. 数据与状态模型

### 6.1 工作态（Working Copy）

前端维护工作态，避免每次输入都直接写盘。

### 6.2 脏状态（Dirty State）

- 任何字段变更后标记为未保存
- 保存成功后清除脏标记

### 6.3 校验状态（Validation State）

- 存储最新校验结果
- 保存前必须无错误

### 6.4 最后更新时间

- 每模块维护最近更新时间用于总览卡片展示

## 7. API 设计（本地）

以下为建议接口（命名可在实现阶段微调）：

- `GET /api/config`
- `POST /api/validate`
- `POST /api/save`
- `POST /api/import`
- `GET /api/export`
- `POST /api/auth/verify`

约束：

- 所有写入类 API 需通过口令会话校验
- `save` 必须执行严格校验门禁

## 8. 错误处理与边界

- 口令错误：停留登录页并提示
- 配置文件不存在/损坏：明确提示并给出修复建议
- 校验失败：
  - 顶部显示错误数量
  - 抽屉中显示字段级定位信息（JSON 模式统一列表）
- 保存失败（IO/权限）：提示失败原因，不丢失工作态

## 9. 非功能要求

- 仅本机访问
- 默认绑定本地回环地址（实现阶段确定）
- 响应式：桌面优先，兼容常见笔记本分辨率
- 可用性：核心操作 2 次点击内可达

## 10. 验收标准

满足以下条件视为 v1 通过：

1. `fastcli web` 能自动启动服务、自动打开浏览器，并在终端打印 URL 与口令
2. 页面必须通过口令输入后才进入后台
3. 后台采用顶部/左侧/主区三段式，首页展示四块总览
4. 可在抽屉中编辑已有条目，支持表单/JSON 双模式
5. JSON 模式支持“点击校验”并统一显示错误列表
6. 保存全部在有错误时必须被阻断
7. 导入支持全量覆盖并有二次确认弹窗
8. 导出能导出当前配置 JSON
9. 保存成功后正确写入 `~/.fastcli/config.json`

## 11. 未来版本方向

- v1.1：优先在 `aliases` 增加删除能力
- v1.2：新增条目能力
- v1.3：导入差异预览与选择性覆盖
